# 个人品牌门户网站需求文档

## 1. 项目概述

### 1.1 项目名称
个人品牌门户网站（Personal Portfolio Website）

### 1.2 项目目标
创建一个现代化的单页个人简历网站，用于展示个人信息、教育背景、专业技能和实习经历，采用深色主题设计，具备良好的用户体验和响应式布局。

---

## 2. 内容层

### 2.1 内容层需求分析

#### 2.1.1 内容类型识别
网站需要展示以下核心内容：
- **个人信息**：姓名、专业、学历、联系方式（生日、电话、邮箱、学校等）、个人简介
- **教育经历**：学校信息、专业信息、时间范围、相关课程、社团活动、毕业设计（可选）
- **专业技能**：核心能力（技能条展示）、工具栈
- **实习经历**：公司信息、职位、时间、工作内容、项目经验
- **个人优势**（可选）：个人优势卡片展示
- **页脚信息**：版权信息、社交媒体链接

#### 2.1.2 内容优先级
- **高优先级**：个人信息、教育经历、专业技能、实习经历
- **中优先级**：页脚信息、返回顶部功能
- **低优先级**：个人优势（可选模块）

#### 2.1.3 内容更新需求
- 内容需要易于更新和维护
- 支持随时添加新的教育经历
- 支持随时添加新的实习经历
- 支持更新技能和工具栈
- 支持更新个人信息和联系方式
- 支持更新个人简介
- 支持添加或删除个人优势模块（如使用）

### 2.2 内容层设计

#### 2.2.1 个人信息内容设计

**姓名展示**
- 大标题展示，响应式字体大小（clamp(2rem,5vw,3rem)）
- 文字阴影效果增强视觉层次

**专业信息展示**
- 副标题格式：专业名称 · 学历
- 使用蓝色（aurora色）突出显示

**联系信息展示**
- 采用2列网格布局（桌面端），单列（移动端）
- 使用图标+文字的组合方式，可包含以下信息类型：
  - 生日：日历图标 + 日期
  - 电话：电话图标 + 电话号码
  - 邮箱：信封图标 + 邮箱地址
  - 学校：毕业帽图标 + 学校名称 · 学院名称
  - 其他联系方式（根据实际需求添加）

**个人简介展示**
- 多行文本描述
- 行高设置为relaxed（1.625）提高可读性

#### 2.2.2 教育经历内容设计

**教育信息卡片内容结构**
- **学校名称**：大标题，加粗显示
- **时间范围**：右侧对齐，蓝色文字
- **专业信息**：专业名称 · 学历 · 学院名称
- **相关课程**（可选）：
  - 标题："相关课程"（小写，字母间距加宽）
  - 内容：课程标签列表，以标签形式展示
  - 标签样式：圆角，半透明背景
- **社团活动**（可选）：
  - 标题："社团活动"
  - 内容：社团活动相关信息，以标签或文本形式展示
- **毕业设计**（可选）：
  - 标题："毕业设计"
  - 内容：毕业设计题目，以标签形式展示

#### 2.2.3 专业技能内容设计

**核心能力模块内容**
- 多个技能项，每个包含：
  - 技能名称（左侧）
  - 百分比数值（右侧，蓝色）
  - 进度条可视化
- **技能展示方式**：
  - 技能名称和百分比并排显示
  - 下方显示进度条，进度条宽度对应百分比数值
  - 进度条使用蓝色到绿色渐变

**工具栈模块内容**
- 多个工具项，每个包含图标+文字
- **工具展示方式**：
  - 使用网格布局展示工具列表
  - 每个工具项包含图标和工具名称
  - 图标使用Font Awesome图标库

#### 2.2.4 实习经历内容设计

**实习经历卡片内容结构**
- **公司名称**：大标题，加粗显示
- **职位**：副标题，次要文字颜色
- **时间**：右侧对齐，绿色文字（aurora-green色）
- **工作内容**：列表形式展示，包含以下内容类型（根据实际情况选择）：
  - **背景**：加粗标题，无"-"符号前缀，描述公司背景或项目背景
  - **岗位职责**：加粗标题，无"-"符号前缀，描述主要工作职责
  - **项目经验**（可选）：
    - 项目名称：加粗标题，单独一行
    - 项目描述：多个子项，每个子项前有绿色"-"符号，描述项目具体内容、成果等
  - **其他工作内容**（可选）：
    - 加粗标题，单独一行
    - 多个子项，每个子项前有绿色"-"符号

**列表项样式规范**
- **加粗标题项**：无"-"符号，文字加粗（strong标签）
- **普通内容项**：绿色"-"符号前缀（text-aurora-green），普通文字
- **间距**：列表项之间使用space-y-2（0.5rem / 8px）

**实习经历数量**
- 支持多个实习经历卡片展示
- 每个实习经历独立成卡片
- 按时间倒序排列（最新的在前）

#### 2.2.5 页脚内容设计

**左侧内容**
- 版权信息：版权符号 + 年份 + 姓名 + 版权声明文字

**右侧内容**
- 社交媒体图标链接：
  - 可包含多个社交媒体平台链接（如GitHub、LinkedIn、微信、微博等）
  - 每个链接使用Font Awesome图标
  - 图标样式：Font Awesome图标
  - 交互：悬停时颜色变为蓝色

#### 2.2.6 个人优势内容设计（可选）

**个人优势卡片内容结构**
- **优势标题**：每个优势卡片包含一个标题
- **优势描述**：多行文本描述，说明该优势的具体内容
- **图标展示**：每个优势卡片可包含一个图标，用于视觉识别
- **卡片布局**：使用网格布局展示多个优势卡片
- **卡片数量**：根据实际需求展示多个优势卡片

## 3. 页面信息架构层

### 3.1 信息架构层需求分析

#### 3.1.1 页面结构需求
- **类型**：单页应用（SPA），所有内容在一个页面展示
- **布局方式**：垂直滚动布局，用户通过滚动浏览不同内容区域
- **导航需求**：需要快速导航到不同内容区域
- **响应式需求**：需要适配不同屏幕尺寸

#### 3.1.2 内容组织需求
- **内容层级**：清晰的标题层级和内容分组
- **视觉引导**：通过时间线、卡片等视觉元素引导用户浏览
- **信息密度**：平衡信息展示和信息可读性

#### 3.1.3 导航结构需求
- **桌面端**：固定顶部导航栏，始终可见
- **移动端**：汉堡菜单，节省屏幕空间
- **导航项**：关于我、教育经历、专业技能、实习经历

### 3.2 信息架构层设计

#### 3.2.1 页面结构设计

**整体布局**
- **类型**：单页应用（SPA）
- **布局**：垂直滚动布局
- **背景**：深蓝色渐变背景（#121E34 到 #1E3A5F）
- **背景固定**：背景图片固定，不随滚动移动
- **粒子效果**：背景包含40个随机浮动的粒子元素

**响应式设计**
- **桌面端**：完整功能展示，横向布局
- **移动端**：适配小屏幕，垂直布局，汉堡菜单
- **断点**：768px（md断点）

#### 3.2.2 导航结构设计

**桌面端导航**
- **位置**：固定顶部
- **样式**：毛玻璃效果（backdrop-filter: blur(10px)）
- **内容**：
  - 左侧：姓名（渐变文字效果）
  - 右侧：导航链接（关于我、教育经历、专业技能、实习经历）
- **交互**：
  - 悬停时链接颜色变为蓝色（aurora色）
  - 滚动时导航栏样式变化（padding和阴影）

**移动端导航**
- **默认状态**：隐藏导航链接，显示汉堡菜单图标
- **展开状态**：点击汉堡菜单后显示垂直菜单
- **菜单项**：与桌面端相同的导航链接

**导航功能**
- **平滑滚动**：点击导航链接平滑滚动到对应section
- **锚点定位**：使用锚点链接（#about, #education, #skills, #internship）

#### 3.2.3 内容组织设计

**内容区域划分**
1. **个人信息头部**（#about）
   - 布局：响应式flex布局
   - 桌面端：头像左侧1/3，信息右侧2/3
   - 移动端：头像和信息垂直排列

2. **教育经历**（#education）
   - 时间线设计：左侧垂直渐变线（蓝色到绿色）
   - 时间点：蓝色圆形点
   - 卡片样式：半透明背景，悬停发光边框

3. **专业技能**（#skills）
   - 布局：2列网格布局（桌面端），单列（移动端）
   - 左侧：核心能力模块
   - 右侧：工具栈模块

4. **实习经历**（#internship）
   - 时间线设计：左侧垂直渐变线（蓝色到绿色）
   - 时间点：绿色圆形点
   - 卡片样式：半透明背景，悬停发光边框

5. **页脚**
   - 布局：响应式flex布局
   - 桌面端：左右分布
   - 移动端：垂直排列

**内容层级关系**
- **一级标题**：section标题（教育经历、专业技能、实习经历）
- **二级标题**：卡片内标题（学校名称、公司名称、核心能力、工具栈）
- **三级标题**：内容分组标题（相关课程、社团活动、毕业设计）
- **内容项**：具体信息展示（列表项、标签等）

**间距设计**
- Section间距：mb-20（5rem / 80px）
- 卡片间距：mb-6（1.5rem / 24px）
- 列表项间距：space-y-2（0.5rem / 8px）
- 网格间距：gap-8（2rem / 32px）

## 4. 功能层

### 4.1 功能层需求分析

#### 4.1.1 导航功能需求
- **需求**：用户需要能够快速导航到不同内容区域
- **场景**：用户点击导航链接，页面平滑滚动到对应区域
- **移动端需求**：在小屏幕上需要节省空间的导航方式

#### 4.1.2 滚动功能需求
- **需求**：页面需要支持平滑滚动
- **需求**：当用户滚动到页面底部时，需要返回顶部功能
- **场景**：用户浏览完内容后，需要快速返回页面顶部

#### 4.1.3 动画功能需求
- **需求**：页面加载和滚动时需要视觉反馈
- **需求**：技能条需要动态展示进度
- **需求**：头像和状态标签需要吸引注意力的动画

#### 4.1.4 响应式功能需求
- **需求**：页面需要在不同设备上正常显示
- **需求**：移动端需要适配的导航和布局
- **场景**：用户在手机、平板、电脑上访问网站

#### 4.1.5 交互功能需求
- **需求**：用户操作需要视觉反馈（悬停效果）
- **需求**：卡片悬停时需要突出显示
- **需求**：社交媒体图标需要交互反馈

### 4.2 功能层设计

#### 4.2.1 导航功能设计

**桌面端导航功能**
- **固定定位**：导航栏固定在页面顶部
- **平滑滚动**：点击导航链接后，页面平滑滚动到对应section
- **偏移调整**：滚动时考虑导航栏高度，偏移80px
- **视觉反馈**：悬停时链接颜色变为蓝色

**移动端导航功能**
- **汉堡菜单**：点击汉堡菜单图标切换菜单显示/隐藏
- **自动关闭**：点击导航链接后自动关闭菜单
- **平滑滚动**：与桌面端相同的平滑滚动功能

**导航栏动态效果**
- **滚动响应**：滚动超过50px时，导航栏padding从py-3变为py-2，添加shadow-lg阴影
- **过渡动画**：过渡时长0.3秒

#### 4.2.2 滚动功能设计

**平滑滚动功能**
- **实现方式**：使用window.scrollTo({ behavior: 'smooth' })
- **导航链接**：点击后平滑滚动到对应section，偏移80px
- **返回顶部**：点击返回顶部按钮后平滑滚动到页面顶部

**返回顶部功能**
- **显示逻辑**：
  - 默认状态：隐藏（opacity-0 invisible）
  - 显示条件：页面滚动超过300px
  - 隐藏条件：页面滚动小于300px
- **位置**：固定右下角（fixed bottom-6 right-6）
- **尺寸**：48px × 48px（w-12 h-12）
- **样式**：圆形按钮，半透明蓝色背景，蓝色边框，向上箭头图标
- **交互**：悬停时背景变深，点击后平滑滚动到页面顶部
- **过渡动画**：过渡时长0.3秒

#### 4.2.3 动画功能设计

**页面加载动画**
- **淡入动画**：section-fade-in
  - 初始状态：透明度0，向下偏移20px
  - 激活状态：透明度1，无偏移
  - 触发条件：滚动到视口85%位置
  - 动画时长：0.8秒

**浮动动画**
- **头像浮动**：animate-float
  - 动画：6秒循环，上下浮动10px
  - 缓动函数：ease-in-out

**脉冲动画**
- **状态标签**：animate-pulse-slow
  - 动画：4秒循环脉冲
  - 缓动函数：cubic-bezier(0.4, 0, 0.6, 1)

**进度条动画**
- **技能条**：滚动到技能区域时触发
  - 初始宽度：0%
  - 目标宽度：根据data-width属性值（83%-90%）
  - 动画时长：1.5秒
  - 缓动函数：ease-in-out
  - 触发条件：技能section进入视口85%位置时

**导航栏滚动动画**
- **滚动效果**：
  - 滚动超过50px时：padding从py-3变为py-2，添加shadow-lg阴影
  - 滚动小于50px时：恢复原始样式
  - 过渡时长：0.3秒

**粒子背景动画**
- **粒子生成**：
  - 数量：40个粒子
  - 大小：随机1-4px
  - 位置：随机分布在页面
  - 动画：使用float动画，随机延迟0-5秒，随机持续时间10-20秒
  - 颜色：半透明蓝色（rgba(58, 159, 238, 0.2)）

**返回顶部按钮动画**
- **显示/隐藏**：
  - 滚动超过300px：显示（opacity-100 visible）
  - 滚动小于300px：隐藏（opacity-0 invisible）
  - 过渡时长：0.3秒
- **悬停效果**：背景从bg-aurora/20变为bg-aurora/30

#### 4.2.4 响应式功能设计

**响应式布局**
- **断点设置**：768px（md断点）
- **桌面端布局**：
  - 完整功能展示，横向布局
  - 导航栏横向排列
  - 个人信息区域：头像左侧1/3，信息右侧2/3
  - 专业技能：2列网格布局
  - 页脚：左右分布
- **移动端布局**：
  - 适配小屏幕，垂直布局
  - 导航栏：汉堡菜单
  - 个人信息区域：头像和信息垂直排列
  - 专业技能：单列布局
  - 页脚：垂直排列

**响应式头像设计**
- **移动端尺寸**：192px × 192px（w-48 h-48）
- **桌面端尺寸**：256px × 256px（w-64 h-64）

**响应式时间线设计**
- **桌面端**：左侧位置（left: 1rem）
- **移动端**：位置调整（left: 0.5rem）

**响应式字体设计**
- **姓名**：响应式字体大小（clamp(2rem,5vw,3rem)）
- **标题**：使用Tailwind响应式类（text-3xl等）

#### 4.2.5 交互功能设计

**悬停效果设计**
- **导航链接**：悬停时文字颜色变为蓝色（aurora色）
- **卡片**：悬停时显示发光边框（aurora-border）
- **社交媒体图标**：悬停时颜色变为蓝色
- **返回顶部按钮**：悬停时背景变深（从bg-aurora/20变为bg-aurora/30）

**平滑滚动设计**
- **导航链接点击**：平滑滚动到对应section，偏移80px（考虑导航栏高度）
- **返回顶部**：平滑滚动到页面顶部
- **滚动行为**：使用behavior: 'smooth'

**移动端菜单交互**
- **切换方式**：点击汉堡菜单图标切换显示/隐藏
- **自动关闭**：点击导航链接后自动关闭菜单
- **过渡效果**：使用Tailwind的hidden类实现显示/隐藏

**卡片交互设计**
- **悬停效果**：卡片悬停时显示发光边框（aurora-border）
- **过渡动画**：所有交互效果使用transition-all duration-300

## 5. 视觉交互层

### 5.1 视觉交互层设计

#### 5.1.1 颜色系统设计

**主色调**
- **Midnight（深蓝）**：#121E34
- **Midnight Light（浅深蓝）**：#1E3A5F
- **Aurora（蓝色）**：#3A9FEE
- **Aurora Green（绿色）**：#00E5A0

**文字颜色**
- **Primary Text（主文字）**：rgba(255, 255, 255, 0.9)
- **Secondary Text（次要文字）**：rgba(255, 255, 255, 0.6)

**背景颜色**
- **Card Background（卡片背景）**：rgba(255, 255, 255, 0.04)
- **Glass Background（玻璃背景）**：rgba(255, 255, 255, 0.04) + backdrop-filter: blur(10px)

**颜色应用规范**
- **主背景**：深蓝色渐变（#121E34 到 #1E3A5F）
- **强调色**：Aurora蓝色用于链接、图标、强调文字
- **辅助色**：Aurora Green用于时间线、时间点、列表符号
